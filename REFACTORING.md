## 📋 Найденные проблемы

### 🔴 Критические проблемы

#### 1. Отсутствие TypeScript
- **Проблема:** Весь код написан на vanilla JavaScript без типизации
- **Влияние:** 
  - Нет автодополнения кода в IDE
  - Труднее поддерживать код
  - Нет проверки типов на ранней стадии
- **Решение:** Перенести на TypeScript или использовать JSDoc

#### 2. Отсутствие модульности
- **Проблема:** Код в inline-скриптах, нет разделения на модули
- **Влияние:** 
  - Код сложно поддерживать
  - Нет возможности реюзинга логики
- **Решение:** Создать модульную структуру (ES6 modules)

#### 3. Hard-coded контент
- **Проблема:** Данные в текстовых файлах (.txt) с простой разметкой — только 5 файлов (`vpr7-1.txt` … `vpr7-5.txt`), покрывающих задачи 1–5 7-го класса. Задачи 6–12 и все задачи 8-го класса данных в .txt-файлах не имеют.
- **Влияние:** 
  - Сложно добавлять/изменять контент
  - Нет валидации данных
  - Неравномерное покрытие
- **Решение:** Использовать JSON с валидацией для всех задач

#### 4. Отсутствие сохранения прогресса
- **Проблема:** localStorage НЕ используется в JS-файлах — поиск `localStorage` по всем `.js` файлам в `vpr7/js/` и `VPR8/js/` дал ноль результатов (верифицировано 18.07.2026)
- **Влияние:** Потеря прогресса при перезагрузке, нет мотивации для продолжения
- **Решение:** Добавить localStorage для сохранения результатов экзаменов и статистики заданий

### 🟡 Важные улучшения

#### 5. Нет PWA-улучшений
- **Проблема:** `manifest.json` присутствует в корне проекта, но `sw.js` отсутствует
- **Влияние:** Нет офлайн-доступа, PWA неполноценный
- **Решение:** Создать sw.js с кэшированием контента (HTML, CSS, JS, шрифты)

#### 6. UI/UX проблемы
- **Проблема:** Визуальная обратная связь только через цвета
- **Влияние:** Может быть недостаточно для слабовидящих
- **Решение:** 
  - Добавить иконки состояний
  - Поддержка экранного ридера
  - Contrast-check

#### 7. Производительность
- **Проблема:** Загрузка всех CSS/JS файлов сразу
- **Влияние:** Медленная начальная загрузка
- **Решение:** 
  - Lazy-load JS модулей
  - Code splitting
  - Tree-shaking

#### 8. Безопасность
- **Проблема:** Нет CSRF защиты (если есть формы)
- **Влияние:** Уязвимость к CSRF атакам
- **Решение:** Добавить CSRF токены

#### 9. SEO и доступность
- **Проблема:** 
  - Мета-теги есть, но не проверяются
  - Alt-тексты могут отсутствовать
  - Нет `<meta name="description">` на страницах заданий
- **Решение:** 
  - Добавить schema.org разметку
  - Проверить доступность (WCAG 2.1)
  - Добавить `<meta name="description">` на каждую страницу

### 🟢 Дополнительные улучшения

#### 10. Тестирование
- **Проблема:** Нет видимой системы тестирования
- **Решение:** 
  - Добавить unit-тесты (Jest/Vitest)
  - E2E-тесты (Playwright)
  - Visual regression testing

#### 11. Локализация
- **Проблема:** Только русский язык
- **Решение:** Добавить английскую версию

#### 12. Аналитика
- **Проблема:** Только Яндекс.Метрика
- **Решение:** Добавить Google Analytics, event tracking для заданий

#### 13. Документация
- **Проблема:** ARCHITECTURE.md может быть неполным
- **Решение:** 
  - Добавить API документацию
  - JSDoc комментарии
  - README для контрибьюторов

#### 14. База заданий
- **Проблема:** Данные в отдельных .txt файлах (только 5 штук)
- **Решение:** 
  - Консолидировать в JSON
  - Добавить генерацию вариантов
  - Добавить тестовые данные для всех заданий

### localStorage для прогресса

```javascript
// js/storage/localStorage.js
const STORAGE_KEY = 'vpr7-progress';

export function saveResult(className, taskId, score) {
  const progress = JSON.parse(localStorage.getItem(STORAGE_KEY) || '{}');
  const key = `${className}-${taskId}`;
  progress[key] = {
    score,
    attempts: (progress[key]?.attempts || 0) + 1,
    lastAttempt: new Date().toISOString()
  };
  localStorage.setItem(STORAGE_KEY, JSON.stringify(progress));
}

export function getProgress() {
  return JSON.parse(localStorage.getItem(STORAGE_KEY) || '{}');
}
```
