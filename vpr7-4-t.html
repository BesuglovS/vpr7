📚 Теория: Составление URL-адреса

🌐 Что такое URL-адрес?

URL (Uniform Resource Locator) — это адрес ресурса в интернете. Он указывает, где находится файл и как к нему получить доступ.

В задании №4 ВПР по информатике нужно уметь составлять URL-адрес по заданным параметрам:

  • Протокол — способ передачи данных (http, https, ftp)
  • Домен (сервер) — имя сервера в интернете
  • Путь — расположение файла на сервере
  • Имя файла — название файла с расширением

📐 Структура URL-адреса

протокол://домен/путь/файл

После домена всегда идёт символ /, затем путь к файлу и имя файла

🔗

Протокол

Протокол — это набор правил, определяющих способ передачи данных. Указывается в начале URL и отделяется от домена символами ://
http://
Веб-страницы (незащищённый)
https://
Веб-страницы (защищённый)
ftp://
Передача файлов
ftps://
Передача файлов (защищённый)

💡 Как запомнить

  • HTTP = HyperText Transfer Protocol (протокол передачи гипертекста)
  • HTTPS = HTTP Secure (безопасный HTTP) — используется шифрование
  • FTP = File Transfer Protocol (протокол передачи файлов)
  • Буква S в конце означает Secure (безопасный/защищённый)
🌐

Домен (сервер)

Домен — это уникальное имя сервера в интернете. Может состоять из букв латиницы или кириллицы (например, example.com или пример.рф).

📝 Примеры доменов:

school.edu — образовательный сайт
docs.info — сайт с документацией
файлы.рф — домен на кириллице

💡 Важно помнить

  • Домен указывается без протокола (без http://)
  • Домен указывается без слеша в конце
  • В задании домен часто называют «сервер»
📁

Путь к файлу

Путь — это последовательность папок (каталогов), в которых находится файл. Каждая папка отделяется символом /.

📝 Примеры путей:

/docs/report.docx — файл в папке docs
/files/download/setup.exe — файл в папке download, которая внутри files

⚠️ Типичные ошибки

  • Забыть символ / между папками
  • Добавить лишний / в начале пути (он уже есть после домена)
  • Перепутать порядок папок (важно: «внутри каталога X каталог Y»)

🔑 Важно: отличие от Windows-путей

  • В URL используется прямой слеш /
  • В Windows используется обратный слеш \
  • Пример: https://site.com/docs/file.pdf — правильно, https://site.com\docs\file.pdf — неправильно!
📄

Имя файла

Имя файла — это название файла с расширением. В URL всегда указывается в конце адреса.

📝 Примеры имён файлов:

отчёт.docx — документ Word
document.pdf — PDF-документ
index.html — веб-страница
📋

Алгоритм составления URL

1

Запишите протокол

Добавьте :// после протокола. Например: https://

2

Добавьте домен (сервер)

Запишите имя сервера после ://. Например: https://example.com

3

Добавьте путь (если есть)

После домена поставьте / и укажите путь к файлу. Например: https://example.com/docs

4

Добавьте имя файла

Поставьте / и запишите имя файла. Результат: https://example.com/docs/report.pdf

✅

Примеры решения задач

📝 Пример 1: Простой URL

Доступ к файлу info.docx, находящемуся на сервере school.edu, осуществляется по протоколу http.

http://school.edu/info.docx

📝 Пример 2: URL с путём

Доступ к файлу readme.txt, находящемуся на сервере docs.info в каталоге files, осуществляется по протоколу https.

https://docs.info/files/readme.txt

📝 Пример 3: URL с глубоким путём

Доступ к файлу задание.pdf, находящемуся на сервере файлы.рф в каталоге 2026 внутри каталога документы, осуществляется по протоколу ftp.

ftp://файлы.рф/документы/2026/задание.pdf

📝 Пример 4: Файл в корне сервера (без пути)

Доступ к файлу index.html, находящемуся на сервере site.ru, осуществляется по протоколу https. Файл находится в корне сервера (не в папке).

https://site.ru/index.html

Обратите внимание: после домена всё равно нужен символ / перед именем файла!

📖

Как читать условие задачи

В заданиях на составление URL используются определённые формулировки. Вот как их правильно понимать:

Формулировка в задаче Что это значит Пример в URL
«на сервере X» Домен = X school.edu
«по протоколу X» Протокол = X https://
«в каталоге X» Одна папка: /X/ /docs/
«в каталоге Y внутри каталога X» Две папки: /X/Y/ /files/download/
«файл F» Имя файла = F report.pdf

💡 Важный момент про порядок папок

  • «в каталоге B внутри каталога A» → путь /A/B/ (сначала внешняя папка, потом внутренняя)
  • Порядок важен! /A/B/ ≠ /B/A/
🔐

Кодирование спецсимволов (дополнительно)

Если в имени файла или пути есть пробелы или специальные символы, они кодируются специальным образом:

Символ Код в URL Пример
пробел %20 my%20file.pdf вместо «my file.pdf»
кириллица %XX%XX %D0%BF%D1%80%D0%B8%D0%BC%D0%B5%D1%80 вместо «пример»

Кириллические домены (например, .рф) технически кодируются с помощью специального алгоритма Punycode: домен пример.рф превращается в xn--e1afmkfd.xn--p1ai. Однако современные браузеры автоматически показывают кириллические домены в читаемом виде, а в заданиях ВПР можно писать их как есть — https://файлы.рф/.

💡 Для заданий ВПР

  • В заданиях ВПР обычно используются имена файлов и путей без пробелов и спецсимволов
  • Кириллические домены (например, .рф) можно писать как есть, кодировать их не требуется
  • Эта информация дана для общего развития и понимания принципов URL

Памятка для запоминания

Часть URL Что это Пример
🔗 Протокол Способ передачи данных http:// https:// ftp://
🌐 Домен Имя сервера school.edu example.com
📁 Путь Папки к файлу docs/files
📄 Файл Имя файла с расширением report.pdf index.html

💡 Золотые правила

  • После протокола всегда идёт ://
  • После домена всегда идёт /
  • Между папками всегда /
  • Путь записывается в порядке вложенности: внешняя папка → внутренняя папка
  • Файл всегда в конце URL
